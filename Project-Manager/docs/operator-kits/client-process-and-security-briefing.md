# Client Process and Security Briefing (Consulting-First Script)

Use this script in fit calls, kickoff calls, and stakeholder updates. This explains how your consulting production process works and how client-sensitive data is protected at each step.

## 1) Opening Positioning (What You Say)

"Our engagement model is consulting-first. My team runs the process for you; your team does not need to self-operate a platform. We use a controlled production workflow with explicit data handling, access controls, and stage-by-stage checkpoints."

## 2) Production Process Overview (Client-Facing)

Explain the process as six stages:

1. Intake and access setup
2. Evidence intake and classification
3. Analysis and structured output drafting
4. Decision packet and executive review
5. Delivery and handoff
6. Retention, archive, and secure offboarding

For each stage, explain three things:

- what happens operationally
- what client data is used
- what security controls apply

## 3) Stage-by-Stage Script (What Happens + Why Secure)

## Stage 1 — Intake and Access Setup

What happens:

- Confirm business objective, scope boundary, and owner approvals.
- Limit data access to minimum required sources.

Security controls:

- Principle of least privilege (minimum permissions only).
- Documented access inventory before work starts.
- No broad "just in case" access.

Client line:

"We begin by limiting access to only what is required for agreed scope, and we document exactly who can access what."

## Stage 2 — Evidence Intake and Classification

What happens:

- Receive or register source materials.
- Classify by data type and sensitivity level.
- Separate source truth from working outputs.

Security controls:

- Sensitive source records are controlled as authoritative evidence.
- Segregation of source truth from derived working artifacts.
- Audit-friendly indexing and traceability.

Client line:

"Your source records remain the controlled truth layer; analysis outputs reference those records without creating unmanaged copies."

## Stage 3 — Analysis and Structured Output Drafting

What happens:

- Perform workflow analysis, reconciliation, and risk triage.
- Draft structured outputs for decisions.

Security controls:

- Work occurs in controlled operator environment.
- Role-limited handling of sensitive artifacts.
- No unnecessary reproduction of raw sensitive records in summaries.

Client line:

"We transform fragmented information into decision-ready outputs while minimizing exposure of raw sensitive detail."

## Stage 4 — Decision Packet and Executive Review

What happens:

- Compile prioritized decisions, risks, and recommended actions.
- Review with authorized stakeholders.

Security controls:

- Audience-scoped materials (only what each stakeholder needs).
- Sanitized outputs for broader distribution.
- Explicit review and approval record.

Client line:

"Decision packets are role-scoped and sanitized so the right people get the right level of detail without overexposing sensitive records."

## Stage 5 — Delivery and Handoff

What happens:

- Deliver final outputs, operating recommendations, and implementation sequence.
- Confirm accepted deliverables and next-step ownership.

Security controls:

- Controlled final package assembly.
- Confirmed recipient list.
- Traceable handoff and acceptance.

Client line:

"Final delivery is controlled and traceable, with acceptance captured so there is clear accountability."

## Stage 6 — Retention, Archive, and Secure Offboarding

What happens:

- Close active work scope.
- Retain required records by policy.
- Archive inactive materials and remove unnecessary active access.

Security controls:

- Retention policy applied by data class.
- Archive separation for inactive records.
- Access removal and closure verification.

Client line:

"At closeout, we reduce active exposure by archiving inactive records and removing access that is no longer required."

## 4) How To Answer Common Client Security Questions

## "Who can see our sensitive data?"

Answer:

"Only assigned roles with scope-based need can access sensitive records. Access is intentionally restricted and documented."

## "Do you duplicate our files everywhere?"

Answer:

"No. We keep source truth controlled and minimize copies. Derived outputs reference source records rather than spreading raw data."

## "How do you prevent scope creep and data sprawl?"

Answer:

"We use stage gates, change control, and data classification controls. New requests are either scoped formally or deferred."

## "What happens to our data when the project ends?"

Answer:

"We execute closeout: required retention, archive separation for inactive assets, and removal of unnecessary active access."

## 5) Client Confidence Summary (Use In Proposals)

"This engagement is delivered as a managed consulting production process with controlled access, traceable evidence handling, role-scoped decision outputs, and secure closeout controls. You get execution and clarity without unmanaged data exposure."

## 6) What Not To Say

Avoid:

- "You can just log in and run this yourself."
- "It is basically self-serve."
- "Everything is open by default."

Use instead:

- "This is operator-managed by our consulting team."
- "We control process execution and access boundaries."
- "You receive structured outputs and guided decisions."

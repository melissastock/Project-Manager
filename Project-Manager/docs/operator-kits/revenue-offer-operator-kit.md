# Revenue Offer Operator Kit (SKU A + SKU B)

This kit is the default operating system for selling and delivering:

- **SKU A**: Clarity Sprint Diagnostic
- **SKU B**: Execution OS Build Sprint

Use this exactly as written until you intentionally version it.

## 1) Offer Summary

## SKU A — Clarity Sprint Diagnostic

- Price: Standard `$3,500`, Fast-track `$5,000`
- Timeline: `7-10 business days`
- Outcome: Decision-ready clarity and a 30-day roadmap

### Required outputs

1. Current-state workflow map (finance/legal/ops)
2. Risk + bottleneck register (with severity scoring)
3. Prioritized decision packet
4. 30-day implementation roadmap
5. Executive readout call

## SKU B — Execution OS Build Sprint

- Price: starting `$12,000` (typical `$12,000-$25,000`)
- Timeline: `90 days` (phased)
- Outcome: Repeatable operating system with measurable execution

### Required outputs

1. Intake + evidence routing workflow
2. Dashboard/tracking baseline
3. Standardized reporting outputs
4. Team cadence + governance loop
5. Handoff docs and enablement

## 2) Qualification Gate (30-minute fit call)

Mark each as `Yes/No`.

- Urgent operating pain exists now (not future-only)?
- Decision owner is on call and accountable?
- Data access available (Drive/Gmail/systems) within 48 hours?
- Team will commit to weekly checkpoint cadence?
- Scope can be constrained to one explicit objective?

### Decision rule

- `4-5 Yes`: proceed to scoped offer.
- `3 Yes`: proceed only with strict boundary conditions.
- `0-2 Yes`: do not sell now; provide readiness blockers.

## 3) Fit Call Script (30 minutes)

## 0:00-5:00 — Frame

1. "What breaks if nothing changes in 30 days?"
2. "Which decisions are currently blocked?"

## 5:00-15:00 — Diagnose

1. "Walk me through intake to decision output."
2. "Where are records fragmented or untrusted?"
3. "Where do legal/financial risks concentrate?"

## 15:00-22:00 — Scope

1. "What is the smallest meaningful win in 10 days?"
2. "Who signs off on outputs?"
3. "What access can start immediately?"

## 22:00-30:00 — Close

1. Recommend SKU A or SKU B.
2. Confirm timeline and package.
3. Set next step: SOW + kickoff date.

## 4) Statement of Work Skeleton (copy/paste)

## A. Objective

Define a single measurable business outcome for this engagement.

## B. Scope In

- Specific workflows covered
- Systems/data sources included
- Stakeholder roles participating

## C. Scope Out

- Explicit exclusions (advisory-only, broad platform rebuild, etc.)

## D. Deliverables

List exact files/outputs and acceptance criteria.

## E. Timeline + cadence

- Kickoff date
- Weekly checkpoint schedule
- Delivery/readout dates

## F. Commercials

- Fixed fee
- Payment schedule
- Change-order process

## 5) SKU A Delivery Runbook (10-day default)

Day 1-2:

- Intake + access verification
- Data/source inventory

Day 3-4:

- Workflow mapping (finance/legal/ops touchpoints)

Day 5-6:

- Risk/bottleneck scoring

Day 7-8:

- Prioritization + implementation roadmap draft

Day 9-10:

- Decision packet finalize
- Executive readout + conversion ask

### SKU A done-definition

- All 5 deliverables complete
- Owner accepted prioritized action sequence
- Clear go/no-go on SKU B

## 6) SKU B Delivery Runbook (90-day phased)

Phase 1 — Design and baseline:

- Process architecture and role mapping
- Intake/evidence workflow design

Phase 2 — Build and operationalize:

- Workflow implementation
- Dashboard and reporting baseline

Phase 3 — Stabilize and handoff:

- Governance cadence in place
- Training + handoff complete

### SKU B done-definition

- Team runs weekly cadence without operator dependency
- Reporting outputs are repeatable
- Decision latency reduced from baseline

## 7) Weekly Client Update Template

Use this structure every week:

1. Outcome this week (1-2 lines)
2. Work completed
3. Risks/blockers
4. Decisions needed (owner + due date)
5. Next-week plan

## 8) Evidence and Data Placement Rules

- **Truth and sensitive records** -> `recovery-core`
- **Processing logic and execution artifacts** -> `execution-finance` / `execution-legal`
- **Client-safe narratives and proof excerpts** -> `narrative-output`
- **Control docs, status, and orchestration** -> `Project-Manager`

Never duplicate legal/financial source truth outside `recovery-core`.

## 9) Conversion Path (SKU A -> SKU B)

At executive readout, ask one explicit decision:

"Do we operationalize this now over a 90-day execution sprint?"

If yes:

1. Confirm scope boundary in writing.
2. Launch SKU B kickoff.
3. Convert roadmap actions into a tracked implementation queue.

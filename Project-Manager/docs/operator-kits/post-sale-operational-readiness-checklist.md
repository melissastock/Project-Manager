# Post-Sale Operational Readiness Checklist (Rapid Start)

Use this checklist immediately after signed SOW + first payment clears.

If any critical item is missing, do not start delivery sprint execution until resolved.

---

## A) Commercial and Scope Controls (Critical)

- [ ] Signed SOW stored and linked.
- [ ] First payment received and reconciled.
- [ ] Scope baseline explicitly locked.
- [ ] Change-order process documented to client.
- [ ] Acceptance criteria documented per deliverable.

---

## B) Client Operating Model (Critical)

- [ ] Client decision owner named.
- [ ] Client day-to-day coordinator named.
- [ ] Response SLAs agreed (standard + urgent).
- [ ] Weekly update day/time agreed.
- [ ] Weekly checkpoint meeting on calendar.

---

## C) Delivery Infrastructure (Critical)

- [ ] Engagement record created in PM-portal.
- [ ] Ticket baseline created (intake, workflow map, risk register, decision packet).
- [ ] Deliverables tracker initialized with owner + due date.
- [ ] Invoices tracker initialized (issued/paid/outstanding).
- [ ] Artifact index prepared for client-safe links.

---

## D) Data and Security Controls (Critical)

- [ ] Data classes identified (legal/financial/medical/IP).
- [ ] Secure storage path defined by client/project/data class.
- [ ] Access roles mapped and approved.
- [ ] Sensitive file handoff method confirmed.
- [ ] Security briefing delivered and acknowledged.

---

## E) Cadence and Reporting (High Priority)

- [ ] Kickoff packet delivered to client.
- [ ] Weekly status packet template configured.
- [ ] Decision log template configured.
- [ ] Escalation path and SLA documented.
- [ ] Readout date tentatively booked.

---

## F) Financial Operations (High Priority)

- [ ] Invoice milestones scheduled.
- [ ] Payment methods confirmed (Stripe/ACH/wire).
- [ ] Invoice recipients confirmed.
- [ ] Late payment follow-up cadence set.
- [ ] Milestone-payment gating policy included in communication.

---

## G) Fast Operational Gaps (Common Missing Items)

If not complete, these are typical sources of drift:

1. Decision owner not consistently available.
2. No explicit response-time agreement.
3. Scope lock not stated in kickoff.
4. Deliverables tracker missing acceptance state.
5. Invoice cadence not communicated up front.

---

## H) Ready-to-Start Gate

Start execution only when all Critical sections are complete:

- Commercial and Scope Controls
- Client Operating Model
- Delivery Infrastructure
- Data and Security Controls

If any critical item is not complete, run a 24-hour blocker resolution loop.

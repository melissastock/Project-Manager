# Last-Mile Workflow: From Start to Payment and Close

Use this as a strict checklist. Do not skip steps.  
If a step is blocked, stop and resolve the blocker before advancing.

---

## Stage 0 — Pre-Sell Setup (one-time)

1. Confirm your two live offers are defined:
   - SKU A: Clarity Sprint Diagnostic
   - SKU B: Execution OS Build Sprint
2. Keep these templates ready:
   - fit call notes template
   - SOW template
   - kickoff checklist
   - weekly client update template
   - final handoff checklist
   - invoice template
3. Define payment terms in writing (default):
   - SKU A: 100% upfront before kickoff
   - SKU B: 50% at kickoff, 30% at midpoint, 20% at final handoff
4. Define hard boundaries:
   - no unpaid kickoff
   - no scope change without change order
   - no final files before final payment clears

---

## Stage 1 — Lead Intake and Qualification

### Goal
Decide fast: proceed now vs not ready.

1. Run 30-minute fit call.
2. Score these 5 gates (Yes/No):
   - urgent pain now
   - decision owner present
   - data/process visibility available
   - willingness for weekly cadence
   - scoped objective is clear
3. Decision:
   - 4-5 yes: proceed to proposal
   - 3 yes: proceed only with strict constraints
   - 0-2 yes: decline or defer
4. Record decision and next action in `Project-Manager` tracker.

Exit criteria:
- Go/No-Go decision documented
- Buyer knows next step and timeline

---

## Stage 2 — Scope, Offer, and Close

### Goal
Convert qualified lead to signed, paid engagement.

1. Send offer summary (SKU A or SKU B).
2. Send SOW with:
   - objective
   - scope in / out
   - deliverables
   - timeline
   - cadence
   - price and payment milestones
   - change-order rule
3. Review SOW live with decision owner.
4. Resolve open items within one revision cycle.
5. Send signature request.
6. Send kickoff invoice based on agreed payment schedule.
7. Confirm funds received before kickoff.

Exit criteria:
- Signed SOW
- Initial payment received
- Kickoff date locked

---

## Stage 3 — Kickoff and Control Setup

### Goal
Establish control so delivery does not drift.

1. Create client workspace records:
   - agreement record
   - owner contacts
   - communication channel
2. Open initial work tickets:
   - intake/data access
   - workflow map
   - risk register
   - decision packet
3. Assign role ownership (single accountable owner per lane).
4. Confirm data access:
   - source systems
   - file locations
   - legal/financial sensitivity flags
5. Lock scope baseline after intake completion.

Exit criteria:
- Intake complete
- Scope baseline locked
- Work queue visible with owners and due dates

---

## Stage 4 — Delivery Execution (Do the Work)

### Goal
Produce promised outputs on schedule.

1. Follow your sprint cadence (SKU A 7-10 days; SKU B phased).
2. Run weekly checkpoints:
   - completed
   - blockers
   - decisions needed
   - next actions
3. Track all new asks:
   - if in scope: queue and prioritize
   - if out of scope: issue change order
4. Keep artifacts placed correctly:
   - truth/sensitive -> `recovery-core`
   - execution work -> `execution-finance` / `execution-legal`
   - client-facing sanitized outputs -> `narrative-output`
5. Keep one rolling “client confidence note” each week:
   - what changed
   - why it matters
   - what is next

Exit criteria:
- Promised deliverables drafted and reviewed internally
- No untracked scope expansion

---

## Stage 5 — Packaging (The Last 10%)

### Goal
Turn work into a decision-ready final package.

1. Assemble final delivery package:
   - executive summary
   - deliverables index
   - decisions made
   - remaining risks
   - 30-day next-step plan
2. Quality check against SOW line by line.
3. Redact/sanitize anything not client-safe.
4. Create handoff map:
   - file name
   - purpose
   - owner
   - action required
5. Prepare readout agenda (45-60 min).

Exit criteria:
- Final package complete and internally approved
- Readout scheduled

---

## Stage 6 — Final Delivery Readout

### Goal
Secure acceptance and transition to payment + next engagement.

1. Run readout in this order:
   - objective recap
   - what was delivered
   - what changed operationally
   - key risks and decisions
   - recommended next sprint
2. Ask for explicit acceptance:
   - “Do you accept this delivery as complete per SOW?”
3. Capture acceptance in writing (email/message/doc signoff).
4. Immediately present continuation option:
   - SKU A -> SKU B conversion
   - SKU B -> retained optimization scope

Exit criteria:
- Written acceptance received
- Next-step proposal presented

---

## Stage 7 — Invoicing and Payment Collection

### Goal
Collect final payment without drift.

1. Issue invoice the same day as accepted readout.
2. Include:
   - SOW reference
   - milestone delivered
   - payment due date
   - payment method details
3. Follow payment rhythm:
   - Day 0: send invoice
   - Day 3: confirm received / remind
   - Day 7: reminder + escalation note
   - Day 10+: call with decision owner
4. Do not release unrestricted editable masters until payment clears.
5. Mark payment status in your tracker.

Exit criteria:
- Funds received and recorded
- Invoice closed

---

## Stage 8 — Closeout (Stop the Spin)

### Goal
Close cleanly so your brain exits the project.

1. Run close checklist:
   - all tickets resolved or parked
   - final package delivered
   - payment closed
   - open risks documented
2. Send closeout memo:
   - outcomes achieved
   - unresolved items
   - recommended next engagement
3. Archive inactive materials:
   - move non-active artifacts to `archive`
4. Capture proof artifact for future sales:
   - before/after
   - measurable gains
   - sanitized case narrative
5. Change project state to `closed`.

Exit criteria:
- Project marked closed
- Proof extracted
- No loose commitments remain

---

## Non-Negotiable Stop Rules

If any condition below is true, pause work immediately:

1. No signed SOW
2. No required payment milestone received
3. Decision owner unavailable for approvals
4. Scope changes requested without change order
5. Sensitive-source handling violates your rules

---

## One-Page Daily Command Sequence

Use this when you feel drift starting:

1. What stage is this project in (1-8)?
2. What is the single exit criterion for this stage?
3. What is blocked?
4. What is the one action that gets to exit criterion today?
5. Do that action first. Nothing else.

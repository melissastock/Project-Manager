# Proposal Package Template (Post-Intake, Post-Estimate)

Use this package **after intake is complete** and **scope/scale fit has been validated**.

This package is consulting-first and assumes you (the operator) run delivery, approvals, and secure handling.

## What is included

1. `01-proposal-cover-letter.md`
2. `02-scope-and-scale-fit-summary.md`
3. `03-estimate-sheet.md`
4. `04-statement-of-work.md`
5. `05-commercials-and-payment-schedule.md`
6. `06-risk-assumptions-and-change-order.md`
7. `07-client-approval-and-signoff.md`
8. `08-send-checklist.md`

## How to use

1. Duplicate this folder into a client-specific path, e.g.:
   - `Project-Manager/docs/client-engagements/<client-or-project>/proposal-package/`
2. Fill sections in order (`01` -> `08`).
3. Do not skip `02` (scope-and-scale fit) or `03` (estimate basis).
4. Send as one package (PDF bundle or shared folder with numbered files).

## Stage gate requirement

Do not send this package until:

- Intake status = `complete`
- Required data access = `verified`
- Scope-and-scale fit = `approved`
- Internal delivery owner = `assigned`

## Data handling reminder

- Sensitive source truth stays in `recovery-core`.
- Proposal package is client-facing and should be sanitized.
- Do not include raw legal/financial source files in the proposal bundle.

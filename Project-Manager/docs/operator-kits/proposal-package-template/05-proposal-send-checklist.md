# Proposal Send Checklist (Post-Intake)

Use this before sending any proposal package.

## Intake and Fit Checks

- [ ] Intake is marked complete.
- [ ] Scope and scale fit summary is complete and approved internally.
- [ ] Estimate is validated against available delivery capacity.
- [ ] Decision owner name and title are confirmed.
- [ ] Security and data-handling expectations are documented.

## Package Contents (all required)

- [ ] `01-scope-and-scale-fit-summary-template.md` (instantiated for client)
- [ ] `02-proposal-cover-template.md` (instantiated for client)
- [ ] `03-statement-of-work-template.md` (instantiated for client)
- [ ] `04-estimate-and-pricing-sheet-template.md` (instantiated for client)

## Commercial and Legal

- [ ] Payment schedule is clearly stated.
- [ ] Invoice trigger conditions are explicitly written.
- [ ] Change-order process is explicitly written.
- [ ] Scope exclusions are explicit and specific.
- [ ] Acceptance criteria are explicit and measurable.

## Security and Process Communication

- [ ] Client-facing process and security briefing is attached or linked.
- [ ] Sensitive data handling model is explained (consultant-operated).
- [ ] Portal/status reporting cadence is defined.

## Final QA

- [ ] Dates, names, pricing, and deliverables are internally consistent.
- [ ] No placeholder text remains.
- [ ] No sensitive source data is embedded in proposal artifacts.
- [ ] Filename conventions are clean and client-ready.

## Send and Log

- [ ] Proposal package sent to decision owner and required stakeholders.
- [ ] Send date logged in Project-Manager tracker.
- [ ] Follow-up date scheduled (within 48 hours).
- [ ] Live review call scheduled.

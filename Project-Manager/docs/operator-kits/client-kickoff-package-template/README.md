# Client Kickoff Package Template (Consulting-Led)

Use this package immediately after sales conversion (signed SOW + kickoff payment received).

This package is client-facing and explains:

- when work starts
- how communication works
- how updates are shared
- how approvals happen
- what the client must provide each week

## Package Contents

1. `01-client-welcome-and-start-confirmation.md`
2. `02-communication-and-response-charter.md`
3. `03-kickoff-meeting-agenda-and-script.md`
4. `04-first-14-days-execution-plan.md`
5. `05-client-update-and-approval-process.md`
6. `06-post-sale-operational-readiness-checklist.md` (internal operator control)

## Recommended Sequence

1. Fill and send `01-client-welcome-and-start-confirmation.md`.
2. Attach `02-communication-and-response-charter.md`.
3. Schedule kickoff and run `03-kickoff-meeting-agenda-and-script.md`.
4. Share `04-first-14-days-execution-plan.md` during kickoff.
5. Set recurring weekly update rhythm from `05-client-update-and-approval-process.md`.
6. Internally run `06-post-sale-operational-readiness-checklist.md` before day-1 execution.

## Trigger Conditions (Do Not Start Without These)

- Signed SOW
- Kickoff payment received
- Decision owner assigned on client side
- Data-access path confirmed

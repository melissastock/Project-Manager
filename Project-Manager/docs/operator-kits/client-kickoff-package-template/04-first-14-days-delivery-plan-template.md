## Day-by-Day / Block Plan

## Days 1-2 — Intake and Access Validation

Objectives:

- confirm source systems and file access
- validate communication channel and decision owner
- register initial artifacts and sensitivity classes

Client responsibilities:

- provide access credentials/permissions
- confirm owner for approvals

Consulting team responsibilities:

- run intake inventory
- open workflow tickets and milestones

Outputs by end of Day 2:

- Intake checklist complete
- Access status table
- Baseline project tracker record

## Days 3-4 — Current-State Workflow Mapping

Objectives:

- map current workflow across legal/financial/ops touchpoints
- identify handoff points and failure points

Client responsibilities:

- validate workflow map accuracy

Consulting team responsibilities:

- produce v1 workflow map
- capture assumptions and confidence notes

Outputs by end of Day 4:

- Workflow map draft
- Assumption register

## Days 5-6 — Risk and Bottleneck Scoring

Objectives:

- score risks by severity and urgency
- identify highest leverage interventions

Client responsibilities:

- confirm impact context for top risks

Consulting team responsibilities:

- build risk register with severity scoring

Outputs by end of Day 6:

- Risk and bottleneck register (prioritized)

## Days 7-8 — Prioritized Action Design

Objectives:

- convert findings into executable recommendations
- align sequence to available capacity

Client responsibilities:

- review action options and constraints

Consulting team responsibilities:

- produce decision options and recommended sequence

Outputs by end of Day 8:

- Decision packet draft
- Implementation sequence options

## Days 9-10 — Readout and Acceptance (SKU A)

Objectives:

- finalize and present outputs
- secure acceptance and next-step decision

Client responsibilities:

- participate in readout
- accept deliverables or request bounded revisions

Consulting team responsibilities:

- deliver final package and capture acceptance

Outputs by end of Day 10:

- Final decision packet
- 30-day implementation roadmap
- Written acceptance status

## Days 11-14 — Transition Window (if continuing)

Objectives:

- launch approved next phase work
- convert actions into tracked execution queue

Outputs by end of Day 14:

- active implementation queue
- confirmed cadence for continuing work

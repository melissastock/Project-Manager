# Project Persona Framework

## Purpose

Define reusable user personas for different project types so delivery, KPI design, and commercialization planning are grounded in real user contexts.

---

## Core Personas By Project Type

### Producer Projects

- **Primary persona:** creative producer/operator
- **Needs:** rapid ideation, packaging flow, feedback loops, asset readiness
- **Key outcomes:** throughput, quality pass rate, launch asset completeness

### Archiavellian Projects

- **Primary persona:** strategy/architecture lead
- **Needs:** decision clarity, cross-project dependency visibility, risk control
- **Key outcomes:** decision velocity, roadmap confidence, governance compliance

### Archive Projects

- **Primary persona:** archive steward
- **Needs:** retention integrity, controlled access, no accidental exposure
- **Key outcomes:** integrity checks, retention completion, access traceability

### Client-Delivery Projects

- **Primary persona:** delivery lead / client stakeholder
- **Needs:** predictable delivery, issue resolution, transparent status
- **Key outcomes:** on-time delivery, SLA attainment, client satisfaction

### Internal-Ops Projects

- **Primary persona:** internal operator/admin
- **Needs:** reliability, clear runbooks, low operational drag
- **Key outcomes:** incident reduction, efficiency, operational compliance

---

## Persona Governance Rules

- Every project must assign at least one primary persona.
- KPI profile must be aligned to the assigned primary persona.
- Launch/commercialization plans must explicitly reference target persona outcomes.
